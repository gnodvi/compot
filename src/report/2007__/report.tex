\chapter {Принцип Гюйгенса и обзор работ}

%\hspace{11.0mm} 
При решении поставленной задачи можно воспользоваться приближением геометрической теории волн, 
т.е. пренебречь понятиями амплитуды и фазы волны. 
Эволюция фронта волны определяется известным физическим принципом Гюйгенса
(см., например, \cite{b:optica}) и не 
требует численного решения дифференциальных уравнений. 
 
Принцип Гюйгенса позволяет находить волновую поверхность в некоторый момент 
времени, если известно ее положение в предшествующий момент. 
 
Для этого каждую точку волновой поверхности в момент времени $t$ следует 
рассматривать как источник вторичных волн. Волновая поверхность каждой вторичной 
волны спустя промежуток времени $\bigtriangleup t$ представляет собой в однородной среде сферу 
радиуса $v \cdot \bigtriangleup t$, где $v$ -- скорость распространения возмущения. 
Искомая волновая поверхность в момент времени  $t + \bigtriangleup t$ -- это 
геометрическая огибающая волновых поверхностей вторичных волн \seepic{f:hfronts}.

\chapter{Распараллеливание на общей памяти через OpenMP}


Библиотека OpenMP (см. \cite{b:openmp}) идеально подходит для разработчиков, желающих 
быстро распараллелить свою
вычислительную программу с большими параллельными циклами. При этом нет необходимости
полностью пересматривать логическую структуру программы. Достаточно добавить в текст
существующей последовательной программы несколько OpenMP-директив.


\medskip
\noindent
{\bf Описание некоторых использованных конструкций:}
\medskip 

\medskip \noindent {\bf \verb|OMP_GET_NUM_PROCS|}\\
Возвращает число процессоров, доступных приложению. 

\medskip \noindent {\bf \verb|OMP_SET_DYNAMIC|}\\ 
Устанавливает состояние флага, разрешающего динамически изменять число нитей. 

\medskip \noindent {\bf \verb|OMP_SET_NUM_THREADS|}\\ 
Позволяет назначить максимальное число нитей для использования в следующей 
параллельной области (если это число разрешено менять динамически). Вызывается 
из последовательной области программы. 

\medskip \noindent {\bf \verb|PARALLEL ... END PARALLEL|}\\ 
Определяет параллельную область программы. При входе в эту область порождаются 
новые (N-1), образуется "команда" из N нитей, а порождающая нить получает номер 0 
и становится основной нитью команды (т.н. "master thread"). При выходе из 
параллельной области основная нить дожидается завершения остальных нитей, и 
продолжает выполнение в одном экземпляре. Предполагается, что в SMP-системе нити 
будут распределены по различным процессорам (однако это, как правило, находится в ведении 
операционной системы). 

\medskip \noindent {\bf \verb|DO ... [ENDDO]|}\\ 
Определяет параллельный цикл. 

\medskip \noindent {\bf \verb|PRIVATE|}\\
Применяется к переменным, которые необходимо сделать приватными. При входе в 
параллельную область для каждой нити создается отдельный экземпляр переменной, 
который не имеет никакой связи с оригинальной переменной вне параллельной области.
